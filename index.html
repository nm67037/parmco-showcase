The PARMCO System

Precision Automated Remote Motor Control & Observation. A closed-loop PID system powered by C (BlueZ/Pigpio) and controlled via Bluetooth.

View Setup View The C Code
Hardware Architecture

Closed-Loop Feedback

The PARMCO system is built for speed and accuracy. Unlike Python implementations, our backend is written in C to handle microsecond-level GPIO interrupts.

The Loop: The Raspberry Pi reads the sensor data via `pigpiod`, calculates the RPM, and runs a PID algorithm to adjust the PWM signal in real-time, maintaining target speed within <1% error.

  • ✓ Low-Latency C Backend
  • ✓ Bluetooth RFCOMM Protocol
  • ✓ PID Error Correction
App Capabilities

Dual Control Modes

The custom Android application connects directly via Bluetooth Sockets to the C server.

Manual Mode

Direct control over the motor's duty cycle. The App sends character commands (e.g., 'f' for faster, 'd' for slower) which the C state machine processes immediately.

Automatic (PID) Mode

Input a "Desired RPM" (e.g., 1200). The C backend calculates the error between the optical sensor reading and your target, applying Proportional and Integral logic to stabilize the motor.

The Logic

C PID Logic

The core C function that handles error correction and auto-tuning.

parmco_server.c GCC 
/* * FUNCTION: update_pid_controller
 * Calculates the difference between Target RPM and Actual RPM,
 * then adjusts the motor speed (PWM) to minimize that error.
 */
void update_pid_controller() {
    if (current_mode != AUTO_MODE || !motor_running) return;

    // 1. Calculate Error
    double error = (double)desired_rpm - (double)rpm_smooth;

    // 2. Calculate Integral (Accumulated Error)
    pid_integral += error;
    if (pid_integral > PID_MAX_INTEGRAL) pid_integral = PID_MAX_INTEGRAL;
    if (pid_integral < PID_MIN_INTEGRAL) pid_integral = PID_MIN_INTEGRAL;

    // 3. Compute Output (P + I + D)
    double output = (PID_KP * error) + (PID_KI * pid_integral);

    // 4. Limit the rate of change (Safety)
    int change = (int)output;
    if (change > MAX_CHANGE_PER_LOOP) change = MAX_CHANGE_PER_LOOP;
    if (change < -MAX_CHANGE_PER_LOOP) change = -MAX_CHANGE_PER_LOOP;

    // 5. Apply to global speed
    speed_percent += change;
    if (speed_percent > 100) speed_percent = 100;
    if (speed_percent < 0) speed_percent = 0;

    // 6. Write to Hardware (PWM)
    hardware_PWM(pi, SPEED_PIN, PWM_FREQ, speed_percent * 10000);
}
